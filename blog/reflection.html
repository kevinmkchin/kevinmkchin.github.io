C++ Reflection: A Short, Practical Guide to Metaprogramming

By Kevin Chin

Reflection, as a subset of metaprogramming, is the ability for a program to inquire about its own structure and code. While some languages like Java and C# provide some limited reflection, most langauges (e.g., C++) have basically no built-in ability to "reflect". Thus, it is necessary to implement our own reflection system when using these languages.

Here are some examples of use cases in order to motivate the implementation of a reflection system. Unreal (as well as many other game engines) use reflection to store information about the structure of game objects (although they also use reflection for many more wonderful features). For example, imagine we create a C++ class for a Car in Unreal. Let's say that this Car has a few properties: a floating point number representing the Car's maximum speed, an integer representing the number of gears the Car has, and a boolean for whether or not the Car is a Race Car. How does the Unreal Editor know about these properties? How does it expose these fields to the editor so that designers can modify these Car properties?
