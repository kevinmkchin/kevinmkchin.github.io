How to switch Windows App to Console App

December 8, 2021

When making a C++ executable on Windows, you have the option of choosing between a Windows app and a Console app. A Windows app is an app with a GUI Window, and a Console app runs in the terminal (e.g. Windows Command Prompt). By switching over a Windows App to a Console App, you will get a command prompt window accompanying your GUI program Window.

Linker Option

Visual Studio or MSVC C++ Compiler

Set the /SUBSYSTEM linker option of the MSVC compiler for your project to CONSOLE. In Visual Studio, you set this option in project properties.

You should also edit the Preprocessor Definitions. Replace the entry “_WINDOWS” with “_CONSOLE”.

MinGW g++

MinGW builds a Console app by default. One must add the -mwindows compiler option to build a Windows app. To go back to a Console app, simply remove -mwindows.

App Entry Point

C/C++ programs have the main function as the program entry point, but Windows apps use WinMain or wWinMain instead. If your program is using WinMain or wWinMain, then you must add the following main function to your code in order to switch over to a Console app.

int main() 
{
    return _tWinMain(GetModuleHandle(NULL), NULL, GetCommandLine(), SW_SHOW);
}

Conclusion

That’s all you need to do to switch a Windows app to a Console app.

For other operating systems like Mac or Linux, you can probably find equivalent guides or figure it out by looking at compiler documentation.
